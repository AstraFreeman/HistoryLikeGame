# Чеклист тестувальника платформи «Історія як гра»

Цей документ призначений для **нетехнічного тестувальника** (вчителя або учня).  
Для кожного пункту: виконайте дію та поставте галочку, якщо результат відповідає очікуванню.

---

## 1. Головна сторінка

Відкрийте `index.html` у браузері.

- [ ] На сторінці видно картки для всіх 8 історичних періодів
- [ ] При натисканні на картку гри — гра відкривається
- [ ] Є кнопка «Культура» — при натисканні вона розкриває додаткові ігри
- [ ] Восьминіг (персонаж у кутку) рухається при натисканні стрілок на клавіатурі
- [ ] Кнопка «Цікавий факт» показує вікно з текстом факту
- [ ] Після проходження хоча б однієї гри — у нижній частині сторінки з'являється статистика (кількість ігор, час тощо)

---

## 2. Налаштування доступності

На **будь-якій** сторінці платформи є іконка шестерні ⚙ у правому нижньому куті.

- [ ] Кліком по шестерні — панель налаштувань відкривається
- [ ] Повторним кліком — панель закривається
- [ ] Перемикач **«Світла тема»** → фон стає білим/світлим
- [ ] Перемикач **«Темна тема»** → фон повертається до темного
- [ ] Кнопка **А+** → текст стає більшим
- [ ] Кнопка **А++** → текст стає ще більшим
- [ ] Кнопка **А** → текст повертається до звичайного розміру
- [ ] **«Зменшити рух»** → анімації та ефекти зупиняються
- [ ] **«Високий контраст»** → межі елементів стають чіткішими та товщими
- [ ] Після **оновлення сторінки** (F5) усі налаштування залишаються збереженими

---

## 3. Навігація

Перевіряйте на кількох різних іграх.

- [ ] На кожній грі є кнопка **«← На головну»** або схожий напис
- [ ] При натисканні цієї кнопки — відкривається головна сторінка `index.html`
- [ ] У шапці гри видно **назву розділу** (наприклад, «Стародавня») та **назву гри** (наприклад, «Вікторина»)

---

## 4. Вікторина

Відкрийте будь-яку гру типу «Вікторина» (наприклад, через картку на головній).

- [ ] Запитання завантажуються — сторінка не є порожньою
- [ ] При введенні **правильної відповіді** — поле або кнопка стає **зеленою**
- [ ] При введенні **неправильної відповіді** — поле або кнопка стає **червоною**
- [ ] **Пароль або підсумок** з'являється лише після того, як усі відповіді правильні
- [ ] При неправильних відповідях пароль **не з'являється**
- [ ] Гра коректно відображається у **світлій темі** (якщо її увімкнено)

---

## 5. Гра з падінням

Відкрийте будь-яку гру, де слова або питання падають зверху вниз.

- [ ] Слова/питання **падають зверху** після початку гри
- [ ] При введенні **правильної відповіді** → питання зникає, рахунок зростає
- [ ] При **неправильній відповіді або пропуску** → кількість «сердець» (життів) зменшується
- [ ] При **0 серцях** → показується екран поразки
- [ ] При **достатній кількості очок** → показується екран перемоги із зображенням

---

## 6. Хронологія (сортування подій)

Відкрийте гру, де потрібно розставити події у правильному порядку.

- [ ] Картки з подіями можна **перетягувати** мишею (або пальцем на телефоні)
- [ ] Кнопка **«Перевірити»** аналізує порядок
- [ ] **Правильний порядок** → картки підсвічуються зеленим
- [ ] **Неправильний порядок** → картки підсвічуються червоним і злегка «трусяться»
- [ ] Кнопка **«Підказка»** → одна або кілька карток мигають, підказуючи правильне місце
- [ ] Після завершення всіх раундів → показується підсумковий екран

---

## 7. Особливі ігри

Відкрийте кожну гру та перевірте основну механіку:

| Гра | Дія | Очікуваний результат | ✓ |
|-----|-----|----------------------|---|
| **Розкопки** (`archaeology`) | Клікнути на клітинку сітки | Клітинка «розкопується», з'являється артефакт або порожнє місце; рахунок зростає при знахідці | |
| **Дипломат** (`diplomat`) | Натиснути кнопку вибору відповіді | Кнопка змінює колір (зелений/червоний); шкали репутації анімуються | |
| **Захист вежі** (`tower-defense`) | Відкрити гру; кликнути на поле для розміщення вежі | Сцена завантажується; вороги рухаються; вежа з'являється на полі | |
| **Кафе-симулятор** (`kafe-simulator`) | Зачекати кілька секунд після відкриття | Клієнти з'являються у черзі; замовлення формується; рахунок оновлюється | |
| **Вежа культури** (`culture-tower`) | Грати кілька ходів | Блоки падають зверху; правильна відповідь прибирає блоки; рівень зростає | |
| **Будівник міста** (`city-builder`) | Кликнути на клітинку сітки | Будівля з'являється на клітинці; ресурси зменшуються | |
| **Галерея культури** (`culture-gallery`) | Натиснути стрілку → або ← | Картинка змінюється; опис оновлюється | |
| **Пасьянс** (`solitaire-v2`) | Перетягнути карту на іншу стопку | Карта переміщається (якщо хід дозволений) | |
| **Бульбашковий шутер** (`bobble-shooter`) | Кликнути у будь-яке місце екрана | Куля вилітає у напрямку кліку; при збігу кольорів бульбашки зникають | |
| **Пазл** (`puzzle-phenikis`) | Перетягнути фрагмент пазлу | Фрагмент переміщається; при правильному складанні — перемога | |
| **Самвидав** (`samvydav`) | Натиснути «Далі» | Текст наступного документа з'являється | |
| **Радіо Свобода** (`radio-svoboda`) | Рухати регулятор частоти | Показник частоти змінюється; з'являється нова «станція» | |

## 8. AR-сторінки (доповнена реальність)

> ⚠️ Для AR потрібен **мобільний телефон з камерою** та роздрукований або відображений на екрані **маркер HIRO**.  
> На комп'ютері без камери — чорне поле, це нормально.

- [ ] AR-сторінка відкривається у браузері мобільного телефону
- [ ] Браузер запитує дозвіл на доступ до камери → дозволити
- [ ] При наведенні камери на маркер HIRO → на екрані з'являється **3D-модель**
- [ ] Кнопка **«← На головну»** є на сторінці та повертає на головну

---

## 9. Статистика та аналітика

- [ ] Відкрийте `docs/analytics.html` після проходження кількох ігор
- [ ] У таблиці відображаються: **назва гри, дата, час, результат**
- [ ] Кнопка **«Завантажити CSV»** → скачується файл із даними
- [ ] Кнопка **«Очистити все»** → з'являється підтвердження; після підтвердження дані видаляються

---

## 10. Критичні сценарії

Перевірте ці сценарії окремо:

| Сценарій | Очікуваний результат | ✓ |
|----------|----------------------|---|
| Відкрити гру → не завершити → натиснути «← На головну» | Головна сторінка відкривається без помилок | |
| Увімкнути **світлу тему** на головній → відкрити будь-яку гру | Гра теж відображається у світлій темі | |
| Налаштувати шрифт **А++** → перейти в іншу гру → повернутись | Збільшений шрифт залишається скрізь | |
| Оновити сторінку (F5) після зміни будь-якого налаштування | Налаштування збережені, нічого не скинулось | |
| Відповісти **неправильно** на всі питання вікторини | Пароль або підсумок **не з'являється** | |
| Дограти гру до **перемоги** | Показується екран перемоги | |
| Дограти гру до **поразки** | Показується екран поразки | |
| Відкрити платформу на **мобільному телефоні** | Сторінка зручно відображається, кнопки легко натискати | |

---

## Додатково: що вважати помилкою

Зверніть увагу та повідомте розробника, якщо:

- Будь-яка сторінка показує **порожній білий або чорний екран** без пояснення
- З'являється текст `undefined`, `null`, `NaN` або `error` у грі
- Зображення замінене **червоним квадратом** із назвою файлу — це означає, що картинка не знайдена
- Кнопка не реагує на натискання
- Текст виходить за межі кнопки або поля
- Гра «зависає» (нічого не відбувається більше 10 секунд без очевидної причини)
