# «Таємниці скарбниці» — Рефакторинг формату тем

## Мета

Замінити поточний формат даних (поняття згруповані за категоріями: соціальна/
економічна/політична/культурна сфери) на формат, де кожна тема відповідає
конкретному **історичному періоду**.

---

```javascript
  { id: 'social',   name: 'Соціальне життя',    color: '#3b82f6', concepts: [...] },

  { id: 'culture',  name: 'Культура і церква',   color: '#f97316', concepts: [...] },
]
```

## Зміни у файлах

| Файл | Зміна |
|------|-------|
| `data/games/coin-sorter-data.js` | Повністю замінити `topics[]` |
| `games/narrative/coin-sorter.html` | Лише `section` у `DATA` + тексти UI (опційно) |
| `data/games-registry.js` | `sectionKey` — прив'язати до відповідного ключа у QUESTIONS_DB |

Ігрова логіка (drag-and-drop, autoPull, checkConversion) **не змінюється** — формат
`topics[]` сумісний: `id`, `name`, `color`, `concepts`.

---

## Кількість понять

Кожна тема потребує щонайменше **12 понять** (= CELL_CAPACITY + 2 буфер).
Якщо понять менше — повторення при роздачі стають занадто частими.

## #реалізувати-потім

### Зменшення максимальної висоти стопки: CELL_CAPACITY 12 → 10

- **Константа:** `var CELL_CAPACITY = 12;` → `var CELL_CAPACITY = 10;`
- **Ефект:** менше монет у клітинці, менш захаращений вигляд, швидше настає конвертація.
- **Ризик:** при FILL_RATIO=0.75 та 12 клітинках, цільове заповнення = `12×10×0.75 = 90` монет.
  Зараз при CELL_CAPACITY=12 це `12×12×0.75 = 108`. Гра стане трохи коротшою — можливо,
  треба буде підняти WIN_SCORE.
- **Теги:** `#реалізувати-потім`
